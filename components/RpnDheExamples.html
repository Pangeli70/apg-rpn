All expressions are evaluated as lowercase. So uppercase characters are converted automatically

Expressions that involve comparison operators return the value 1 for TRUE and 0 for FALSE

Some operators and expressions are incompatible with the calc route that uses the querystring

Trigonometric functions require angle values operands in radians.

In order of precedence

  • ^ : Elevation to exponent (eg. 3^3=27)
  • / : Division (eg. 3250/250=13)
  • * : Multiplication (eg. 20*0.1=2)
  • % : Modulo operator between integers. Returns the remainder of the division. (eg. 3250%250) use %25 instead of % in url.
  • - : Difference (eg. 4-4=0).
  • + : Sum (eg. 2+2=4 ).
  • # : Round to the specified number of significative figures (eg. pi#5 ) use %23 instead of # in url.
  • > : Greater than comparison (*)
  • < : Lesser than comparison (*)
  • >= : Greater or equal comparison (*)
  • <= : greater or equal comparison (*)
  • = : Comparison (*)

*) Not compatible with the calc route that uses urlencoded querystring. Use the interactive page.

  • y : variable value
  • z : variable value

WARNING: if the expression contains variables, in order to get a valid result, those must be provided! Use the following routes:

  • variables
  • variables ranges

(... : means any computed expression that returns a value):

  • abs(...) : absolute value
  • floor(...) : round to nearest lower integer
  • ceil(...) : round to nearest upper integer
  • round(...) : round to nearest upper integer
  • sqrt(...) : square root
  • log(...) : base ten logarithm
  • deg(...) : convert value from radians to degrees
  • rad(...) : convert value from degrees to radians
  • sin(...) : trigonometric sine of angle in radians
  • cos(...) : trigonometric cosine of angle in radians
  • tan(...) : trigonometric tangent of angle in radians
  • asin(...) : trigonometric arcsine in radians
  • acos(...) : trigonometric arccosine in radians
  • atan(...) : trigonometric arctangent in radians

  • pi : PI Greek = 3.1415923...
  • e[n]: number in scientific notation where n means *10^n (eg. 5.15e3 = 5150, 5e-3 = 0.005)
  • true / false: boolean values will be converted respectively to 1 and 0.

Using the /rpn/calc route

Use parameter '?expr={expression}' in the query string to execute the calculation. If the parameter is not provided a default expression is used.

Use parameter '&res={value}' in the query string to test against the expected result. If the comparison is truthy returns 1 or 0 instead.

Include the parameter '&log=false' to the query string to suppress the inclusion in the logs in the result.
